PUG

Pug - это препроцессор HTML и шаблонизатор, который был написан на JavaScript для Node.js.

Плюсы

  1. c pug мы получаем меньше кода, по сравнению с html
  2. работает emmet как и в чистом html
  3. форматирование кода происходит с помощью отступов
  4. отсутствуют закрывающиеся теги
  5. объявление id и class-ов сходно с написанием классов в css
  6. атрибуты тегов можно записывать через запятую или без нее, также можно разбивать на строки
  1. можно вставлять куски html непосредственно в pug
  2. можно использовать js для генерации html
  3. можно подключать миксины и шаблоны
  4. можно инклудить готовые куски кода
  5. при использовании pug можно хорошо распаралелить разработку проекта, что ведет за собой ускорение верстки

Начало разработки на pug.

Прежде чем начать кодить на pug, необходимо настроить Sublime или другой редактор.

В случае с Sublime, достаточно через Package Control установить пакет Pug.

Собрать сборку с pug можно самому (но это отдельная история) или выкачать существующую

Теги

По умолчанию текст в начале строки представляет собой HTML-тег. Теги с отступом вкладываются, создавая древовидную структуру HTML.

pug
ul
	li Item A
	li Item B
	li Item C
html
<ul>
	<li>Item A</li>
	<li>Item B</li>
	<li>Item C</li>
</ul>

Pug так сам определяет самозакрывающиеся теги

pug
img
input(type="text")
br
html
<img/>
<input type="text">
<br>

Непосредственно в Pug можно вставлять элементы в HTML синтаксисе

pug
p This is plain old <em>text</em> content.
html
<p>This is plain old <em>text</em> content.</p>

Атрибуты

pug
a(href='google.com') Google
|
|
a(class='button' href='google.com') Google
|
|
a(class='button', href='google.com') Google
html
<a href="google.com">Google</a>
<a class="button" href="google.com">Google</a>
<a class="button" href="google.com">Google</a>

Многострочные атрибуты

pug
input(
  type='checkbox'
  name='agreement'
  checked
)
html
<input type="checkbox" name="agreement" checked="checked" />

Pug поддерживает switch case

pug
- var friends = 10
case friends
  when 0
    p you have no friends
  when 1
    p you have a friend
  default
    p you have #{friends} friends
html
<p>you have 10 friends</p>

Циклы

  • Удобно использовать циклы для заполнения страници одинаковым контентом
  • создание списка соц иконок
  • создание меню
  • создание breadcrumbs
  • создание списка постов
  • и т.д.
pug
ul
  each val, index in ['zero', 'one', 'two']
    li= index + ': ' + val
html
<ul>
  <li>0: zero</li>
  <li>1: one</li>
  <li>2: two</li>
</ul>
pug
- var n = 0;
ul
  while n < 4
    li= n++
html
<ul>
  <li>0</li>
  <li>1</li>
  <li>2</li>
  <li>3</li>
</ul>
pug
- for (var x = 0; x < 3; x++)
  li item
html
<li>item</li>
<li>item</li>
<li>item</li>

Комментарии

Существуют различные комментарии: те, которые будут отображаться после компиляции, и те, которые пропадут.

pug
// just some paragraphs
//- will not output within markup
p foo
p bar
html
<!-- just some paragraphs-->
<p>foo</p>
<p>bar</p>

Блочные коментарии

pug
body
  //-
    Comments for your template writers.
    Use as much text as you want.
  //
    Comments for your HTML readers.
    Use as much text as you want.
html
<body>
  <!--Comments for your HTML readers.
Use as much text as you want.-->
</body>

Инклюды (Includes)

Pug имеет возможность вставки содержимого одного файла в другой файл Pug.

pug
include pug/common/_head
  block content
    include pug/common/_header
    main#main(role="main")
      .container
        h1 Home page
    include pug/common/_footer
html

Переменные

В Jade, как и в любом препроцессоре и шаблонизаторе есть переменные. Переменные мы можем объявить с помощью 2 способов. Первый способ объявить непосредственно в нашем файле, второй способ — вынести их в отдельный файл.

Переменные могут объявляться как до их вызова, так и после, также можно подключить переменные в отдельном файле, и с помощью include подключить к нашему шаблону.

Переменные мы можем использовать где угодно и где они нам могут понадобится.

Миксины

С момощью миксин можно не повторять каждый раз одинаковый код, а только менять наполнение блоков

mixin bgRetina(obj)
  .bg-retina
    span(data-srcset="images/small-"+obj.imgName+".jpg, images/small-"+obj.imgName+"@2x.jpg 2x")
    span(data-srcset="images/"+obj.imgName+".jpg", data-media="(min-width: 768px)")

На странице мы прописываем только маленикий кусок кода, а для каждой картинки мы только меняем ее название

+bgRetina({
  imgName: 'slide-01'
})
